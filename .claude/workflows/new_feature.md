\# Workflow: Nova Feature — VanPro



\## Objetivo

Guiar o sistema multiagente desde a proposta até a entrega de uma nova feature do VanPro.



\## Etapas



\### 1. Definição (Mentor → Cérebro)

\- Mentor descreve a feature desejada

\- Cérebro lê MEMORY.md, PROJECT\_CONTEXT.md e DECISIONS\_LOG.md

\- Cérebro confirma entendimento e pergunta se há contexto adicional



\### 2. Pesquisa (Cérebro → Researcher)

\- Researcher investiga abordagens, bibliotecas e trade-offs

\- Entrega relatório com recomendação ao Cérebro



\### 3. Arquitetura (Cérebro → Architect)

\- Architect propõe estrutura da feature

\- Define arquivos afetados, protegidos, complexidade e dependências

\- Cérebro apresenta proposta ao Mentor para aprovação



\### 4. Implementação (Cérebro → Coder)

\- Coder implementa exatamente o que foi aprovado

\- Marca blocos com // \[CODER] INÍCIO e // \[CODER] FIM

\- Entrega ao Revisor antes de qualquer outra etapa



\### 5. Revisão (Cérebro → Reviewer)

\- Reviewer analisa lógica, segurança, padrões e completude

\- Resultado: APROVADO / APROVADO COM RESSALVAS / REPROVADO



\### 6. Segurança (Cérebro → Security)

\- Security analisa riscos da implementação

\- Obrigatório se houver autenticação, dados sensíveis ou integrações externas



\### 7. Testes (Cérebro → Tester)

\- Tester valida fluxo feliz, edge cases e regressão

\- Resultado: PASSOU / PASSOU COM RESSALVAS / FALHOU



\### 8. Documentação (Cérebro → Docs)

\- Docs registra o que foi feito, arquivos afetados e instruções de uso



\### 9. Memória (Cérebro → Memory Keeper)

\- Memory Keeper estrutura bloco de atualização do MEMORY.md

\- Cérebro cola no arquivo e encerra a sessão



\### 10. Aprovação Final (Cérebro → Mentor)

\- Cérebro apresenta resumo completo da entrega

\- Mentor aprova ou solicita ajustes
